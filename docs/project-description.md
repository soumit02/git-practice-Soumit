#  Detailed Project Description

## Objective
The primary goal of this repository is to practice and master fundamental version control workflows using Git and GitHub. Instead of just learning commands, this project involves practically managing a structured codebase.

## Key Git Workflows Practiced
During the development of this project, the following Git operations were successfully executed:
1. **Repository Initialization:** Setting up a local `.git` environment and linking it to a public GitHub repository.
2. **Branching & Merging:** Creating isolated branches (e.g., `feature/calculator`, `feature/error-handling`) to develop new features safely without breaking the main code.
3. **Commit History Management:** Writing clean, descriptive, and atomic commit messages to track changes effectively.
4. **Ignoring Files:** Utilizing `.gitignore` to prevent unnecessary files (like `__pycache__`) from being tracked.
